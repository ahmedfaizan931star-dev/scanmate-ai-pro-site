Data Safety Summary

A plain-language summary to help keep Google Play disclosures consistent with the Privacy Policy.

Important: This page is a guide for transparency. Your final Play Console Data Safety answers must match the actual APK behavior, SDKs, permissions, and any future changes.

Current app data practices summary

AreaSummary
AccountNo account is required for core features.
Core documentsScans, OCR text, exports, and document metadata are intended to remain local on the device unless the user chooses to export/share or use optional online AI.
Optional AIIf the user adds a Gemini API key and submits text/prompts, that submitted content may be sent to Google Gemini API over the internet for processing.
CameraCamera access is used for document and QR scanning.
BiometricBiometric/fingerprint access is used for device-level protection where supported.
Developer sale of dataThe developer does not sell personal or sensitive user data.

Suggested Play Console wording to verify

  • If you declare “No data collected,” ensure optional Gemini AI requests are disabled, absent, or not counted as collection under Google’s current definitions. If optional AI sends user text to a third-party provider, answer carefully.
  • For camera access, explain it is used for scanning documents and QR content.
  • For user files/documents, explain local processing and user-controlled export/share.
  • For optional AI, disclose third-party processing if users can send document text/prompts externally.
  • If you add analytics, ads, crash reporting, login, backend, or cloud sync later, update this page, the Privacy Policy, and Play Console before release.
